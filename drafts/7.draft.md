Students built a custom agent wrapper using Playwright that bypassed dynamic selector issues through:

- **Visual screenshot capture** of UI landmarks before interaction attempts
- **Relative grounding algorithms** that map visual positions to DOM coordinates
- **Landmark detection systems** that identify persistent visual elements (buttons, forms, navigation) rather than transient IDs

This approach reduced selector fragility by 70%+ compared to traditional methods, enabling stable automation across multiple SaaS platforms.

## 3. State Machine Resilience via Checkpointing

**TL;DR Summary:** Multi-step agent failures waste expensive LLM tokens, prompting implementation of checkpoint recovery that reduced token costs by approximately 80% during development cycles.

The team's agents routinely failed on intermediate steps—often step 4 of 8-step workflows—forcing complete restarts that consumed significant LLM API tokens. With each retry burning $0.50-$2.00 in compute costs, development velocity suffered.

Their solution implemented a state machine with periodic checkpointing:

```
[Step 1] → [Step 2] → [Step 3] → [Checkpoint A]
                    ↘             ↗
                     [Step 4] → [Checkpoint B]
```

Key components included:

- **State serialization** at each major workflow transition
- **Progress tracking** via JSON state files stored in cloud storage
- **Resume logic** that validates checkpoint integrity before continuing
- **Token budget monitoring** to prevent runaway costs during retries

This checkpointing system reduced token expenditure by up to 80% during iterative development, making long-running agent workflows economically viable.
