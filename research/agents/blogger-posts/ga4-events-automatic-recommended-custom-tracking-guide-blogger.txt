TITLE: GA4 Events: Automatic, Recommended, and Custom Events Explained

<p><em>Originally published at <a href="https://www.praveentechworld.com/blog/ga4-events-automatic-recommended-custom-tracking-guide">https://www.praveentechworld.com</a>.</em></p>
<p>I spent my first month with GA4 confused about events. There were events that showed up automatically, events I was supposed to create myself, and events Google recommended but did not enforce. I did not know which ones to use or when.</p>
<p>After working through it, the breakdown is simpler than it seems. GA4 has three categories of events: automatic, recommended, and custom. Each serves a different purpose.</p>
<h2>Automatic Events: The Freebies</h2>
<p>Automatic events fire without any configuration. Install the GA4 tag and these start working:</p>
<ul>
<li><code>page_view</code> — every page load</li>
<li><code>session_start</code> — each new session</li>
<li><code>first_visit</code> — first time a user visits your site</li>
<li><code>user_engagement</code> — page is in focus for 10+ seconds</li>
</ul>
<p>These cover the basics. You get page view counts and session data without touching any code. If you have enhanced measurement enabled (which you should), you also get <code>scroll</code>, <code>outbound_click</code>, <code>site_search</code>, <code>video_start</code>, and <code>file_download</code> events.</p>
<p>Automatic events cannot be modified. You cannot add custom parameters to them or change how they fire. They are the foundation of your GA4 data, and everything else builds on top of them.</p>
<p>If you are not seeing these events, check your enhanced measurement settings in Admin &gt; Data Streams &gt; your web stream. For more on getting the basics right, see the <a href="https://www.praveentechworld.com/blog/google-analytics-4-page-tracking-how-it-works">GA4 page tracking guide</a>.</p>
<h2>Recommended Events: The Template</h2>
<p>Recommended events are predefined events that Google suggests for specific industries and use cases. They have standardized names and parameters that unlock built-in reports.</p>
<p>If you run an ecommerce site, Google recommends events like:</p>
<ul>
<li><code>view_item</code> — someone viewed a product</li>
<li><code>add_to_cart</code> — product added to cart</li>
<li><code>purchase</code> — completed transaction</li>
<li><code>remove_from_cart</code> — product removed</li>
</ul>
<p>If you run a content or lead generation site, Google recommends:</p>
<ul>
<li><code>generate_lead</code> — form submission</li>
<li><code>sign_up</code> — account creation</li>
<li><code>share</code> — content shared via social</li>
</ul>
<p>Using recommended events has two advantages. First, GA4's built-in reports automatically populate with your data when you use the correct event names. Second, your data is standardized, meaning you can compare your metrics against industry benchmarks without manual mapping.</p>
<p>The key difference from custom events: you are supposed to implement these yourself via code or Google Tag Manager. They do not fire automatically. But the names and parameters are predefined by Google.</p>
<p>For more on using GA4 data effectively, the <a href="https://www.praveentechworld.com/blog/how-to-use-google-analytics-4-to-improve-your-content-strategy">GA4 content strategy guide</a> covers how to structure your measurement approach.</p>
<h2>Custom Events: Your Own Rules</h2>
<p>Custom events are events you define yourself. You choose the name, you choose the parameters, you choose when they fire. This is where GA4 becomes truly flexible.</p>
<p>I use custom events for things specific to my site:</p>
<ul>
<li><code>article_read</code> — fires when a reader scrolls past 75% of an article</li>
<li><code>tool_used</code> — tracks when someone opens a free tool on my site</li>
<li><code>newsletter_signup</code> — fires on email form submission (since I want extra parameters beyond the default)</li>
</ul>
<p>Custom events have one rule: do not use names that conflict with automatic or recommended events. For example, do not create a custom event called <code>page_view</code> because that name is reserved. Google ignores custom events that clash with reserved names.</p>
<p>Each custom event can carry up to 25 parameters. Parameters are extra data points attached to the event, like article title, tool name, or signup source. These parameters make your reports useful instead of just showing "an event happened."</p>
<p>If you are new to setting up custom events, the <a href="https://www.praveentechworld.com/blog/how-to-use-google-analytics-4-to-improve-your-content-strategy">GA4 content strategy guide</a> walks through practical examples.</p>
<h2>How to Choose the Right Event Type</h2>
<p>Here is a simple decision framework:</p>
<ul>
<li>If Google tracks it automatically with enhanced measurement, use the automatic event</li>
<li>If Google has a recommended event for your action (like <code>purchase</code> or <code>generate_lead</code>), use the recommended event name</li>
<li>If your action is unique to your site, create a custom event</li>
</ul>
<p>Most sites should rely primarily on automatic and recommended events. Custom events fill the gaps for site-specific tracking. Do not create custom events for things Google already tracks.</p>
<h2>Implementing Events with Google Tag Manager</h2>
<p>The easiest way to implement recommended and custom events is through Google Tag Manager (GTM). Add a GTM tag that fires on the action you want to track, and configure the event name and parameters in the tag.</p>
<p>For example, to track a button click as a recommended <code>generate_lead</code> event:</p>
<p>1. Create a new tag in GTM</p>
<p>2. Choose "GA4 Event" as the tag type</p>
<p>3. Set event name to <code>generate_lead</code></p>
<p>4. Add parameters like <code>value</code> and <code>currency</code></p>
<p>5. Set the trigger to fire when someone clicks the button</p>
<p>GTM handles the code injection, so you do not need to edit your site's source files. This is why GTM and GA4 are usually set up together.</p>
<h2>Common Mistakes</h2>
<p><strong>Creating custom events for automatic events.</strong> I see people creating custom <code>scroll</code> events when enhanced measurement already tracks scrolls. Check what enhanced measurement covers before writing any custom code.</p>
<p><strong>Wrong event names.</strong> A typo in an event name means that data ends up in a separate event instead of the report you expected. GA4 does not validate event names, so a <code>purchse</code> event (missing the 'a') creates a new event instead of populating the purchase report.</p>
<p><strong>Too many custom events.</strong> I started with 12 custom events and quickly realized I was tracking things I never looked at. Start with 3 to 5 custom events aligned with your core goals. Add more only when you have a specific question that existing events cannot answer.</p>
<p><strong>Missing parameters.</strong> An event without parameters tells you something happened but not what or where. Every custom event should include at least a <code>name</code> or <code>id</code> parameter that identifies the specific item or action.</p>

<h2>Frequently Asked Questions</h2>
<p><strong>Q: Can I modify automatic events?</strong></p>
<p>A: No. Automatic events fire with predefined names and parameters. You cannot change them. If you need different data, create a custom event that fires alongside the automatic event.</p>
<p><strong>Q: How many custom events can I create?</strong></p>
<p>A: GA4 allows up to 500 unique event names per property. In practice, most sites should use far fewer. More events mean more noise, not more insight.</p>
<p><strong>Q: Do recommended events work without GTM?</strong></p>
<p>A: Yes. You can send recommended events directly from your website code using the <code>gtag()</code> function or the Google Analytics Measurement Protocol. GTM just makes management easier.</p>
<p><strong>Q: What happens if I use the wrong event name?</strong></p>
<p>A: GA4 stores the event as-is. It does not check for typos or correct invalid names. The data still appears in your reports but not in the expected location. Always test new events in the Realtime report before relying on them.</p>
<p><strong>Q: Can I rename an event after creating it?</strong></p>
<p>A: No. GA4 does not support renaming events. You can delete unused events from the admin panel, but the historical data under the old name remains. Test event names carefully before deploying them in production.</p>
