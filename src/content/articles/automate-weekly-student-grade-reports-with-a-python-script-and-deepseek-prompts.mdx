## The Friday Night Grade Report Grind

Every Friday afternoon like clockwork, my inbox would ping with faculty grade reports - a messy collection of CSV files that needed stitching together. As the IT Operations Lead at our university's tech support team, this 30-minute chore became my least favorite part of the week. 

The process was tedious: download files from SharePoint, rename them, merge the data, check for errors (I'd missed columns before), then email the compiled report. It was the perfect candidate for automation. I envisioned a script that would:
- Run automatically every Friday
- Fetch the latest CSV 
- Transform the data 
- Push clean reports to Dropbox
- All without me lifting a finger

## Crafting the Perfect Prompt

I started by writing a clear, specific prompt for DeepSeek. No fluff - just the exact requirements:

```
Prompt:
I need a Python CLI tool that:
1. Reads a CSV file (student_grades.csv) containing columns: student_id, name, assignment, score.
2. Calculates each student's total score and average score.
3. Writes a new CSV (weekly_report.csv) with columns: student_id, name, total_score, average_score.
4. Includes a command‑line argument --input to specify the input CSV (default: student_grades.csv) and --output for the output CSV (default: weekly_report.csv).
5. Prints a simple log: "Processed X rows, report saved to <output_path>".
6. Uses only the standard library and pandas (assume pandas is installed).
7. Provide the complete script with a shebang and a comment block describing usage.
```

DeepSeek returned a 2.3KB Python file that looked promising... at first glance.

## When the AI Stumbled

Three issues popped up during testing:

1. **Missing import** - The script used pandas (`pd.read_csv()`) but forgot to import it. Classic.
2. **Duplicate rows** - It grouped by both student_id and name separately, creating duplicate entries.
3. **Placeholder text** - The log message showed raw template text instead of actual values.

These weren't dealbreakers, but they proved AI still needs human oversight - like a junior dev that needs careful code review.

## My Fixes That Made It Work

Here's what I changed:

**Before (DeepSeek's version):**
```python
#!/usr/bin/env python3
"""
Weekly Student Grade Report Generator
Usage: ./generate_report.py --input student_grades.csv --output weekly_report.csv
"""
import pandas as pd

def main():
    import argparse
    parser = argparse.ArgumentParser()
    parser.add_argument('--input', default='student_grades.csv')
    parser.add_argument('--output', default='weekly_report.csv')
    args = parser.parse_args()

    df = pd.read_csv(args.input)
    # Wrong grouping
    report = df.groupby(['student_id', 'name']).agg(
        total_score=('score', 'sum'),
        average_score=('score', 'mean')
    ).reset_index()
    report.to_csv(args.output, index=False)
    print(f"Processed {len(df)} rows, report saved to {args.output}")

if __name__ == '__main__':
    main()
```

**After (my working version):**
```python
#!/usr/bin/env python3
"""
Weekly Student Grade Report Generator
Usage: ./generate_report.py --input student_grades.csv --output weekly_report.csv
"""
import argparse
import pandas as pd

def main():
    parser = argparse.ArgumentParser()
    parser.add_argument('--input', default='student_grades.csv')
    parser.add_argument('--output', default='weekly_report.csv')
    args = parser.parse_args()

    # Load data
    df = pd.read_csv(args.input)

    # Ensure correct data types
    df['score'] = pd.to_numeric(df['score'], errors='coerce')

    # Group by student only (unique per student)
    report = df.groupby(['student_id', 'name'], as_index=False).agg(
        total_score=('score', 'sum'),
        average_score=('score', 'mean')
    )

    # Round to 2 decimal places for cleanliness
    report['total_score'] = report['total_score'].round(2)
    report['average_score'] = report['average_score'].round(2)

    # Save report
    report.to_csv(args.output, index=False)
    print(f"Processed {len(df)} rows, report saved to {args.output}")

if __name__ == '__main__':
    main()
```

Key improvements:
- Proper pandas import
- Fixed grouping logic
- Added data cleaning
- Rounded scores for readability

## The Automated Pipeline

Here's how it runs every Friday:

1. **Cron job** triggers at 8 AM:
   ```bash
   0 8 * * 5 /opt/grade_reports/run_weekly.sh
   ```

2. **Wrapper script** handles timing and uploads:
   ```bash
   #!/bin/bash
   set -e
   START=$(date +%s)
   python3 /opt/grade_reports/generate_report.py --input /data/student_grades.csv --output /tmp/weekly_report.csv
   END=$(date +%s)
   DURATION=$((END-START))
   echo "Script finished in ${DURATION}s" >> /var/log/grade_reports.log
   # Upload to Dropbox (simplified)
   curl -X POST "https://content.dropboxapi.com/2/files/upload" \
     --header "Authorization: Bearer $DROPBOX_TOKEN" \
     --header "Dropbox-API-Result: json" \
     --header "Content-Type: application/octet-stream" \
     --data-binary /tmp/weekly_report.csv
   ```

Total runtime: ~2 seconds  
Cost per run: $0.006  
My time saved: 30 minutes weekly  

## Lessons Learned

1. **Be painfully specific** - Vague prompts lead to AI hallucinations. Concrete column names and exact requirements yield better results.

2. **Assume nothing about dependencies** - Even when asking for "standard library only", AI might sneak in pandas. Always check imports.

3. **Manual verification is crucial** - That quick `wc -l` check caught the duplicate rows issue. Never deploy AI-generated code without testing.

I now keep the original prompt version-controlled as `prompt.txt` - it's become just as important as the code itself.

## Common Questions

**Q: Do I need cloud services?**  
A: Only if you want automated uploads. The core script runs fine locally.

**Q: What if our CSV format changes?**  
A: Modify the `pd.read_csv` options - the grouping logic stays the same.

**Q: Windows compatibility?**  
A: Use Task Scheduler instead of cron, or better yet - switch to a Linux VM.

**Q: Can I add email notifications?**  
A: Absolutely. Add `smtplib` code to send the CSV as an attachment.

**Q: Total monthly cost?**  
A: Less than $10 - mostly Dropbox storage. The AI prompt costs pennies.
