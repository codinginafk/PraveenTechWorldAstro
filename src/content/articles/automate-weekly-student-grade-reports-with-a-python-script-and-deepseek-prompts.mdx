## The Friday Night Grade Report Grind

Every Friday afternoon, like clockwork, I’d find myself buried under a pile of faculty grade reports - a chaotic mix of CSV files that needed to be pieced together. As the IT Operations Lead at our university's tech support team, this 30-minute task quickly became my least favorite part of the week. 

The whole process felt like a chore: downloading files from SharePoint, renaming them, merging the data, checking for mistakes (I’ve definitely overlooked columns before), and then emailing the final report. It was screaming for a bit of automation. So, I thought, why not create a script that would:

- Run automatically every Friday
- Fetch the latest CSV 
- Transform the data 
- Push clean reports to Dropbox
- All without me lifting a finger

## Crafting the Perfect Prompt

I started by drafting a clear and specific prompt for DeepSeek. I aimed for clarity and precision—no fluff, just the essentials:

```
Prompt:
I need a Python CLI tool that:
1. Reads a CSV file (student_grades.csv) containing columns: student_id, name, assignment, score.
2. Calculates each student's total score and average score.
3. Writes a new CSV (weekly_report.csv) with columns: student_id, name, total_score, average_score.
4. Includes a command‑line argument --input to specify the input CSV (default: student_grades.csv) and --output for the output CSV (default: weekly_report.csv).
5. Prints a simple log: "Processed X rows, report saved to <output_path>".
6. Uses only the standard library and pandas (assume pandas is installed).
7. Provide the complete script with a shebang and a comment block describing usage.
```

DeepSeek produced a Python file that looked promising at first glance... but there were some hiccups.

## When the AI Stumbled

During testing, I ran into three main issues:

1. **Missing import** - The script used pandas (`pd.read_csv()`) but forgot to actually import it. Classic rookie mistake.
2. **Duplicate rows** - It incorrectly grouped by both student_id and name separately, which led to duplicate entries.
3. **Placeholder text** - The log message displayed raw template text instead of actual values.

While these weren’t complete dealbreakers, they showed that AI still needs a bit of human oversight—kind of like a junior developer who requires a good code review.

## My Fixes That Made It Work

Here’s what I changed:

**Before (DeepSeek's version):**
```python
#!/usr/bin/env python3
"""
Weekly Student Grade Report Generator
Usage: ./generate_report.py --input student_grades.csv --output weekly_report.csv
"""
import pandas as pd

def main():
    import argparse
    parser = argparse.ArgumentParser()
    parser.add_argument('--input', default='student_grades.csv')
    parser.add_argument('--output', default='weekly_report.csv')
    args = parser.parse_args()

    df = pd.read_csv(args.input)
    # Wrong grouping
    report = df.groupby(['student_id', 'name']).agg(
        total_score=('score', 'sum'),
        average_score=('score', 'mean')
    ).reset_index()
    report.to_csv(args.output, index=False)
    print(f"Processed {len(df)} rows, report saved to {args.output}")

if __name__ == '__main__':
    main()
```

**After (my working version):**
```python
#!/usr/bin/env python3
"""
Weekly Student Grade Report Generator
Usage: ./generate_report.py --input student_grades.csv --output weekly_report.csv
"""
import argparse
import pandas as pd

def main():
    parser = argparse.ArgumentParser()
    parser.add_argument('--input', default='student_grades.csv')
    parser.add_argument('--output', default='weekly_report.csv')
    args = parser.parse_args()

    # Load data
    df = pd.read_csv(args.input)

    # Ensure correct data types
    df['score'] = pd.to_numeric(df['score'], errors='coerce')

    # Group by student only (unique per student)
    report = df.groupby(['student_id', 'name'], as_index=False).agg(
        total_score=('score', 'sum'),
        average_score=('score', 'mean')
    )

    # Round to 2 decimal places for cleanliness
    report['total_score'] = report['total_score'].round(2)
    report['average_score'] = report['average_score'].round(2)

    # Save report
    report.to_csv(args.output, index=False)
    print(f"Processed {len(df)} rows, report saved to {args.output}")

if __name__ == '__main__':
    main()
```

Some key improvements I made:
- Proper pandas import
- Fixed the grouping logic
- Added data type cleaning
- Rounded scores for better readability

## The Automated Pipeline

So, how does it all run every Friday? Here’s the setup:

1. **Cron job** triggers at 8 AM:
   ```bash
   0 8 * * 5 /opt/grade_reports/run_weekly.sh
   ```

2. **Wrapper script** manages timing and uploads:
   ```bash
   #!/bin/bash
   set -e
   START=$(date +%s)
   python3 /opt/grade_reports/generate_report.py --input /data/student_grades.csv --output /tmp/weekly_report.csv
   END=$(date +%s)
   DURATION=$((END-START))
   echo "Script finished in ${DURATION}s" >> /var/log/grade_reports.log
   # Upload to Dropbox (simplified)
   curl -X POST "https://content.dropboxapi.com/2/files/upload" \
     --header "Authorization: Bearer $DROPBOX_TOKEN" \
     --header "Dropbox-API-Result: json" \
     --header "Content-Type: application/octet-stream" \
     --data-binary /tmp/weekly_report.csv
   ```

The total runtime? Just about 2 seconds.  
Cost per run? Roughly $0.006.  
Time saved weekly? A solid 30 minutes! 

## Lessons Learned

1. **Be painfully specific** - Vague prompts can lead to AI misunderstandings. Providing concrete column names and clear requirements really helps.

2. **Assume nothing about dependencies** - Even when asking for "standard library only," AI might sneak in pandas. Always double-check your imports.

3. **Manual verification is crucial** - A quick `wc -l` check helped me catch the duplicate rows issue. Never trust AI-generated code without thorough testing.

I now keep the original prompt version-controlled as `prompt.txt`—it’s become just as vital as the code itself.

A: Not unless you want automated uploads. The core script runs perfectly fine on your local machine.

**Q: What about Windows compatibility?**  
A: You can use Task Scheduler for that, or better yet, switch to a Linux VM for easier cron integration.

**Q: Can I add email notifications?**  
A: Absolutely! You can implement `smtplib` to send the CSV as an email attachment.
