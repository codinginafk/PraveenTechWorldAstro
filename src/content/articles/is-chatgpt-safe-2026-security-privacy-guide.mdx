## My $2,300 OpenAI Bill Was a Wake-Up Call - Here's How I Fixed It

The moment I saw that $2,300 charge from OpenAI, I knew something was wrong. When I dug into the logs, the problem became painfully clear: we had four developers, seven microservices, and absolutely no visibility into which ones were burning through our token budget. [Our comparison of AI tools](/blog/chatgpt-vs-claude-vs-gemini-which-ai-assistant-is-best-in-2026) shows how costs can vary dramatically between providers. The worst offender? An agent I thought was doing simple classification work was actually sending 12,000 tokens per request - someone had copy-pasted a prototype's entire context window and never cleaned it up.

That's when I stopped treating AI costs like a utility bill and started managing them like production infrastructure. I built a dashboard that:

1. Intercepts every API call through middleware
2. Logs tokens, models, and endpoints to SQLite
3. Shows costs broken down by agent, user, and hour

The result? We cut our monthly bill by 60% in 30 days - not by switching to cheaper models, but by actually seeing where the waste was happening.

### The Solution: A Simple Proxy That Reveals Everything

Here's the hard truth: every AI agent consumes tokens, but most teams have no idea how many. My solution was a transparent HTTP proxy that:

- Sits between your services and LLM providers
- Logs every request's model, tokens, latency, and cost
- Stores everything in SQLite for easy querying
- Adds less than 5ms overhead

```python
from flask import Flask, request, jsonify
import requests, sqlite3, json, time

app = Flask(__name__)
UPSTREAM = "https://api.openai.com"

def log_to_db(model, prompt_tokens, completion_tokens, latency_ms, agent_id, cost):
  db = sqlite3.connect("token_burn.db")
  db.execute("INSERT INTO token_log VALUES (?, ?, ?, ?, ?, ?, datetime('now'))",
    (model, prompt_tokens, completion_tokens, latency_ms, agent_id, cost))
  db.commit()

@app.route("/v1/<path:subpath>", methods=["POST", "GET"])
def proxy(subpath):
  headers = {k: v for k, v in request.headers if k.lower() != "host"}
  t0 = time.time()
  resp = requests.request(request.method, f"{UPSTREAM}/v1/{subpath}",
    headers=headers, json=request.get_json(silent=True), stream=False)
  latency = int((time.time() - t0) * 1000)
  agent = request.headers.get("X-Agent-ID", "unknown")
  data = resp.json()
  usage = data.get("usage", {})
  pt = usage.get("prompt_tokens", 0)
  ct = usage.get("completion_tokens", 0)
  model = data.get("model", "unknown")
  cost = (pt * 0.00015 + ct * 0.0006) / 1000  # gpt-4o rates
  log_to_db(model, pt, ct, latency, agent, cost)
  return jsonify(data), resp.status_code
```

### What We Found (And Fixed)

After a month of tracking, the data revealed some painful truths:

1. **Duplicate Agents**: Two separate services were doing identical classification work, accounting for 40% of our costs. Merging them saved $920/month.

2. **Context Overload**: One agent was sending 48,000 tokens of chat history for simple yes/no questions. Adding `max_history: 5` cut its costs by 80%.

3. **Peak Hour Waste**: Our 2 PM cron job reprocessed all daily data instead of just new changes, creating a $180/hour spike.

### Who This Works For

This approach makes sense if:
- You have multiple developers/services using AI
- Your monthly bill exceeds $100
- You want to identify waste without switching models

It won't help if:
- You use providers that don't return token counts
- Developers bypass the proxy
- You have a single, simple use case

### Alternatives Worth Considering

1. **Helicone**: Open-source proxy with hosted dashboard (free for 100K requests)
2. **LangSmith**: Full-featured but requires LangChain ($99/month)
3. **Manual Logging**: I tried this first - it lasted 3 days before being abandoned

### Key Takeaways

1. **Start simple**: Even basic token logging reveals waste
2. **Look for ratios**: Prompt-to-completion ratios >10:1 indicate bloat
3. **Watch peaks**: Uneven usage often means inefficient scheduling
4. **Tag everything**: Use `X-Agent-ID` headers to track sources
