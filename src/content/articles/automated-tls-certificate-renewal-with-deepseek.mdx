---

I automated TLS certificate renewal using a Python script generated by DeepSeek, but the initial version nearly caused a production outage. The script had missing dependencies, timing issues with cron, and some completely made-up logic. After adding manual safety checks and debugging the AI's hallucinations, I finally turned it into a reliable CLI tool I could trust.

## Why I needed automation

Every 90 days, I'd spend my Sunday afternoon manually renewing TLS certificates across three Nginx servers. The process went like this:
1. SSH into each server
2. Backup the current configs
3. Run `certbot renew`
4. Restart services
5. Repeat for each domain

It was tedious, error-prone, and I constantly worried about missing an expiration date. My monitoring system would alert if any certs expired, but I wanted a solution that just worked automatically at 2 AM without waking me up at 3 AM.

## Getting DeepSeek to write the first draft

I fed DeepSeek this detailed prompt (the more context, the better the output):

```python
I need a Python script that:
1. Takes a list of domains (via CLI or config file)
2. Checks current cert expiration using openssl or certbot
3. Renews certs expiring within 30 days
4. Validates new certs
5. Gracefully restarts Nginx
6. Logs everything with timestamps
7. Has a dry-run mode
8. Uses argparse for CLI options
9. Handles errors gracefully
10. Provides a summary of successes/failures
```

The initial 284-token response (about 2.3KB of code) looked promising. It had proper argparse setup, logging, and even included a JSON config template. I saved it as `tls_renewal.py` and tested it locally - worked fine in about 12 seconds. The dry-run mode produced sensible output. Everything seemed ready for production.

## When the script failed spectacularly

I set up a cron job to run weekly at 2 AM:
```
0 2 * * 0 /usr/bin/python3 /opt/scripts/tls_renewal.py --domains=example.com,api.example.com
```

At 3 AM, I got a panicked call. The script failed immediately with `ImportError: No module named 'cryptography'`. Looking deeper, I found three critical issues:

1. The `check_expiry` function was completely fake - it always returned "45 days from now" so renewals never triggered
2. Missing Python dependencies (`cryptography`) on production servers
3. Inconsistent Nginx service names across servers (`nginx` vs `nginx.service`)
4. Permission issues with log files

The AI had given me a convincing prototype that fell apart under real-world conditions.

## How I fixed it

I spent about 4 hours turning the prototype into production-ready code with three key improvements:

### 1. Real dependency management
Added a `requirements.txt` and automatic dependency checking:
```python
REQUIRED_LIBS = ["cryptography"]

def ensure_dependencies():
    missing = []
    for lib in REQUIRED_LIBS:
        try:
            __import__(lib)
        except ImportError:
            missing.append(lib)
    if missing:
        logging.critical(f"Missing libraries: {missing}")
        sys.exit(1)
```

### 2. Actual certificate checking
Replaced the placeholder with real certbot/openssl calls:
```python
def get_cert_expiry(domain):
    try:
        # Try certbot first
        result = subprocess.run(["certbot", "certificates", "--domains", domain], ...)
        # Parse actual expiry date from output
        return parse_date(result.stdout)
    except Exception:
        # Fall back to openssl
        result = subprocess.run(["openssl", "x509", "-noout", "-enddate", ...], ...)
        return parse_openssl_date(result.stdout)
```

### 3. Robust safety features
Added:
- Proper error handling
- Configurable Nginx service names
- JSON output for monitoring
- Better logging
- Dry-run improvements

## The final product

The production-ready script now:
1. Checks for required dependencies upfront
2. Uses real certificate expiration checks
3. Handles different Nginx service names
4. Provides detailed logging
5. Outputs machine-readable summaries
6. Has comprehensive dry-run mode

To run it:
```bash
# One-time test
python3 tls_renewal.py --domains=example.com --verbose

# Weekly cron job
0 2 * * 0 /usr/bin/python3 /opt/scripts/tls_renewal.py >> /var/log/cron_tls_renewal.log 2>&1
```

## Key lessons learned

1. **AI needs very specific instructions** - My initial prompt didn't explicitly ask for real certificate checking, so the AI used a placeholder
2. **Production is different from development** - Missing dependencies and environment differences will break things
3. **Safety features are mandatory** - Dry-run mode and good logging saved me hours of debugging
4. **Monitoring is still needed** - I set up alerts to catch any failures in the automated process
5. **Cost-effective** - The AI prompt cost $0.04 and dependencies $0.02 - far cheaper than my time doing manual renewals

The AI gave me a 80% solution, but that last 20% required human judgment and real-world experience. For another example of this approach, see how I [automated server health checks](/blog/automated-server-health-checks-with-deepseek).
