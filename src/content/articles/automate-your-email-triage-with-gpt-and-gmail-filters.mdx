## Why This Matters (real-world problem this solves)

My inbox used to be a battlefield. I’d wake up to 87 new emails, but only 5 actually needed my attention by lunch. The rest? Newsletters, marketing blasts, social updates, and automated reports. I’d spend the first 30 minutes of my day just cleaning, not working.

You’ve been there. That low-hanging email anxiety that follows you. But here’s the thing: most of that noise is predictable. If it’s predictable, you can automate the cleanup. That’s exactly what I did, and it’s saved me about an hour every single workday.

## Step-by-Step Guide

### 1. Set Up Your API Key

First, you need an OpenAI API key. Head over to the OpenAI website and sign up. Once you’re in, go to the API keys section and create a new secret key. Copy this immediately. You won’t see it again.

Think of this key as a password. It lets your script talk to the AI brain. I store mine in a secure note on my phone for quick access.

### 2. Write the Core Apps Script

Don’t worry, you don’t need to be a coder. I’ve written the basic script for you. Go to [script.google.com](https://script.google.com) and create a new project. Paste this code in:

```javascript
function labelAndArchive() {
  var apiKey = 'YOUR_API_KEY_HERE';
  var label = GmailApp.getUserLabelByName('AI-Processed');
  var threads = GmailApp.search('is:unread -label:AI-Processed', 0, 50);

  for (var i = 0; i < threads.length; i++) {
    var message = threads[i].getMessages()[0];
    var subject = message.getSubject();
    var body = message.getPlainBody().substring(0, 500); // First 500 chars

    var prompt = "Classify this email into one category: 'Action Required', 'FYI', or 'Archive'. Email subject: " + subject + ". Body: " + body;

    var response = UrlFetchApp.fetch('https://api.openai.com/v1/chat/completions', {
      'method': 'post',
      'headers': {
        'Authorization': 'Bearer ' + apiKey,
        'Content-Type': 'application/json'
      },
      'payload': JSON.stringify({
        "model": "gpt-3.5-turbo",
        "messages": [{"role": "user", "content": prompt}],
        "temperature": 0.1
      })
    });

    var result = JSON.parse(response.getContentText());
    var classification = result.choices[0].message.content.trim();

    if (classification === 'Archive') {
      threads[i].addLabel(label);
      threads[i].archive();
    } else if (classification === 'FYI') {
      threads[i].addLabel(label);
    }
  }
}
```

Replace `YOUR_API_KEY_HERE` with your actual key. This script looks at your first 50 unread emails and asks the AI to sort them.

### 3. Create Your Gmail Filters

Now, we make sure the script only runs on the right emails. Open Gmail. Click the search bar, then "Show search options."

I start with a safe, broad filter. Try something like:
`from:(*newsletter* OR *no-reply* OR *marketing*) -label:AI-Processed`

Click "Create filter," then "Apply the label: AI-Processed." Check "Skip the Inbox (Archive it)." This catches obvious junk. The script will then process any of these that slip through and haven’t been touched.

### 4. Run It and Test

Go back to your Apps Script. Click the "Run" button. Authorize it the first time. It’ll take a minute. Check your Gmail. You should see some emails get archived and labeled.

I tested this on my spammy promotions folder first. It worked so well, I expanded it to social notifications next.

### 5. Schedule It to Run Daily

Don’t do this manually. In Apps Script, click the clock icon (Triggers). Add a new trigger. Set it to run `labelAndArchive` daily, say at 7 AM. Now your triage happens while you sip your morning coffee.

## Tips for Best Results

1.  **Start small.** Test on one unimportant label or email category first. Don’t point this at your VIP emails immediately.
2.  **The cost is low.** GPT-3.5 is incredibly cheap. Processing 100 emails costs less than a quarter.
3.  **Refine your prompt.** The script’s magic is in the instruction. Tweak the "Classify this email..." part to match your needs.
4.  **Check the "FYI" label.** This is your new, clean inbox. These are emails you can read later without panic.
5.  **Add an emergency override.** I added this line to the script to protect emails from my boss: `-from:boss@company.com`. Never auto-archive the boss.

## Frequently Asked Questions

**1. Is this secure? Will it read all my emails?**
The script only accesses emails matching your search query. It sends a small snippet to OpenAI's API. They don’t store this data by default. You can review their privacy policy, but for most people, this is a safe, controlled trade-off for massive time savings.

**2. What about email costs? OpenAI isn't free.**
It’s incredibly inexpensive. The GPT-3.5 model is priced at about $0.0002 per 1K tokens. A typical email snippet is maybe 200 tokens. You’d need to process over 500,000 emails to hit $10 in costs. It’s negligible.

**3. Can I use this for more than just archiving?**
Absolutely. Change the script’s action. Instead of `archive()`, you could have it add a specific label like `WeeklyDigest` or even draft a simple auto-reply. The classification step is what opens up all the automation possibilities.
