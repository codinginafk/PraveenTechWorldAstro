## My Spreadsheet Cleaning Sidekick: How [DeepSeek](/articles/automated-tls-certificate-renewal-with-deepseek) AI Saved Me 3 Hours a Week

Let's be real--building this tool came from a place of frustration. Every Monday morning, I was confronted with the same exhausting task: cleaning 12-15 Excel files packed with all sorts of data messiness. After enduring this for months, I decided to turn to DeepSeek AI for help. The result? A Python CLI tool that cleans files in under 5 seconds each! Although much of the code was generated by AI, I still had to tackle some very human challenges along the way.

### The Spreadsheet Horror Show

Imagine this recurring nightmare:
- Merged cells that leave pandas gasping for air
- Dates formatted in three different ways within the same column
- Random empty rows disrupting the data flow
- "N/A", "NULL", and "---" all pretending to be missing data

I just wanted a simple command:
```bash
cleanxls --input messy_file.xlsx --output clean_file.csv
```
And voilà, my sanity restored!

### First Attempt: When AI Meets Reality

I kicked things off with a straightforward prompt to DeepSeek:
```python
"""
Prompt: Write a Python CLI tool using Click that:
1. Takes an Excel file as input
2. Handles merged cells by unmerging and filling values downward
3. Standardizes dates to YYYY-MM-DD
4. Replaces all null markers (N/A, NULL, ---) with empty strings
5. Outputs a clean CSV
Include error handling for corrupt files.
"""
```

The AI delivered a 127-line script utilizing:
- `pandas` for the heavy lifting
- `click` for the command line interface
- `openpyxl` to handle merged cells

But here's where I hit my first reality check: the script assumed all files were in .xlsx format. Our field teams still send .xls files from their ancient systems.

### Where Things Went Wrong

Three main issues popped up:

1. **Format Blindness**: The script crashed on .xls files, resulting in:
   ```python
   AttributeError: 'Workbook' object has no attribute 'merged_cells?
   ```

2. **Overeager Date Conversion**: It mistakenly tried to convert invoice numbers like "Order #2024-1001" into dates. Not what I needed!

3. **Silent Failures**: If the files were password-protected, it would quietly output empty CSVs. Not cool!

### The Fixes That Made It Work

**1. Handling Different File Types**  
I refined my prompt to include:
```python
"""
Add support for both .xls (use xlrd) and .xlsx (openpyxl). 
First detect file extension, then use the appropriate engine.
"""
```

**2. Smarter Date Handling**  
I added a sanity check:
```python
if re.match(r'^\d{1,4}[-\/]\d{1,2}[-\/]\d{1,4}$', str(cell_value)):  
    # Only then attempt date conversion
```

**3. Better Error Management**  
Now it gracefully handles:
- Password-protected files
- Corrupt files (using magic number checks)
- Permission issues

### The Final Product

Using this tool looks like this:
```bash
cleanxls --input messy.* --output_dir ./clean \
         --date-format "%Y-%m-%d" \
         --null-markers "NULL,N/A,---"
```

And the output keeps me in the loop:
```
Processing 15 files...
✔ Success: 12 files (avg 4.2s/file)
✖ Failed: 3 files (password protected)
Clean files saved to ./clean/
```

I even added some bonus features:
- Wildcard support for batch processing
- Customizable null markers
- A progress bar because no one likes waiting!

### Lessons Learned

1. **AI Misses Real-World Messiness**  
   DeepSeek didn't account for our legacy .xls files. Always specify your exact file formats.

2. **Better Prompts Beat Debugging**  
   Instead of fixing code, I refined my prompt:
   ```python
   """
   Modify to handle both .xls and .xlsx by:
   1. Checking file extension
   2. Using xlrd for .xls
   3. Using openpyxl for .xlsx
   """
   ```
   This got me most of the way there faster than manual coding.

3. **Never Trust Silent Operations**  
   The original script would overwrite files without warning. Now it asks:
   ```python
   if os.path.exists(output_path):
       raise click.Confirm("Overwrite--)
   ```

### The Magic Prompt

For anyone looking to recreate this, here's the final prompt that worked:
```python
"""
Write a Python CLI tool using Click that:
1. Accepts multiple input files via wildcard (e.g., 'data/*.xls*')
2. Detects file extension to use correct engine (xlrd for .xls, openpyxl for .xlsx)
3. Unmerges cells by filling values downward
4. Converts only valid dates to YYYY-MM-DD (skip strings with letters)
5. Replaces configurable null markers (default: NULL,N/A,---)
6. Validates output directory exists
7. Shows progress bar with tqdm
8. Skips password-protected files with a warning
Output clean CSVs with '_clean' suffix.
"""
```

## FAQ  

**1. Can this handle massive Excel files'**  
Yes! Just add `--chunksize 10000` to process in batches. The initial version tried to load everything at once--not ideal for memory.

**2. How do I add my own cleaning rules'**  
The `clean_column()` function is designed for easy extension. The AI actually structured this part quite well.

**3. Why Click instead of argparse'**  
DeepSeek defaulted to Click, and I kept it for the nice auto-generated help screens. Less work for me!

**4. Can this run automatically'**  
Absolutely! I set it up with a cron job to process files in our SFTP folder every Monday at 6 AM.

**5. What about Google Sheets'**  
That's next on my to-do list using `gspread`. The authentication part looks a bit tricky, though.
