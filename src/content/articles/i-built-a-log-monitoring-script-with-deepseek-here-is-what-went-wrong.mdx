## Where the AI Code Failed

The first red flag appeared when I ran the script on a clean VM:


Other issues I discovered:
1. The sliding-window logic reset after each check instead of maintaining state
2. The webhook payload contained non-serializable datetime objects
3. OpenCode's version completely omitted the webhook functionality
4. Neither version properly handled missing dependencies

The cost was minimal (about $0.0018 for DeepSeek's API call), but the debugging time added up.

## How I Fixed the AI's Code

I used DeepSeek's output as a starting point and made these key improvements:

1. **Dependency checks** - Added clear error messages when `colorama` or `requests` were missing
2. **Proper windowing** - Fixed the deque logic to maintain error counts across checks
3. **JSON fixes** - Converted datetimes to ISO strings before webhook calls
4. **Graceful degradation** - Made colored output optional when `colorama` isn't available
5. **Better daemonizing** - Swapped to `python-daemon` for proper PID file handling
6. **Robust logging** - Added log rotation to prevent disk bloat

The final script runs in 0.32s for initial load and uses minimal CPU (under 2% on a 2-core server).

## Key Lessons Learned

1. **AI is like a junior dev** - Great for first drafts but needs careful review
2. **Test in clean environments** - What runs on your dev machine might fail elsewhere
3. **Prompt specifically** - My detailed prompt helped, but I still needed to add "check for missing packages"
4. **Log everything** - Adding proper logging saved hours of debugging
5. **Cost vs. quality** - DeepSeek's output was more useful despite the small cost

The fixed script has been running smoothly on my production servers for a month now. It's not enterprise-grade (you'd want proper rate limiting and TLS verification for that), but it solves my immediate problem.
