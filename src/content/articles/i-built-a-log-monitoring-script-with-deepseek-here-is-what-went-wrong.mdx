Let me walk you through my journey from messy log files to a working solution - complete with the exact prompt you can use for your own experiments!

## The Problem That Kept Me Up at Night

Every morning, I found myself scrolling through `/var/log/nginx/error.log`, trying to spot new issues. My servers were generating hundreds of errors overnight, and I needed a way to know when things went south quickly. The pattern was clear: if I saw more than three errors in a 5-minute window, I wanted an alert. 

What I needed was a lightweight [Python script](/articles/automate-weekly-student-grade-reports-with-a-python-script-and-deepseek-prompts) that could:
- Tail the log file in real-time
- Count errors in rolling windows
- Send a ping to my Slack whenever the thresholds were crossed
- Run smoothly on any Ubuntu box without heavy dependencies

After a week of coding manually, I thought, "Why not ask AI for help -  So, I turned to DeepSeek to generate the complete workflow, hoping to learn how to prompt AI for real automation tasks.

Surprisingly, DeepSeek returned 560 tokens (about 3.8KB) of polished code. It had colored output, used `requests` properly, and even included daemon functionality. For comparison, OpenCode's output was more compact but completely messed up the sliding-window logic.

## Where the AI Code Failed

The first red flag popped up when I ran the script on a clean VM:

```python
Traceback (most recent call last):
  File "log_monitor.py", line 87, in <module>
    ImportError: No module named 'colorama?
```

Here were some other issues I stumbled upon:
1. The sliding-window logic reset after each check rather than maintaining state.
2. The webhook payload had non-serializable datetime objects.
3. OpenCode's version entirely omitted the webhook functionality.
4. Neither version properly handled missing dependencies.

The cost for using DeepSeek was minimal (around $0.0018 for the API call), but the debugging time? That really added up.

## How I Fixed the AI's Code

I took DeepSeek's output as a starting point and made these key improvements:

1. **Dependency checks** - I added clear error messages when `colorama` or `requests` were missing.
2. **Proper windowing** - I fixed the deque logic to maintain error counts across checks.
3. **JSON fixes** - I converted datetimes to ISO strings before the webhook calls.
4. **Graceful degradation** - Made colored output optional if `colorama` isn't available.
5. **Better daemonizing** - Switched to `python-daemon` for proper PID file handling.
6. **Robust logging** - Added log rotation to avoid disk bloat.

The final script runs in 0.32 seconds for initial load and uses minimal CPU (under 2% on a 2-core server).

## Key Lessons Learned

1. **AI is like a junior dev** - Great for first drafts but really needs careful review.
2. **Test in clean environments** - What works on your dev machine might fail elsewhere.
3. **Prompt specifically** - My detailed prompt helped, but I still had to add "check for missing packages."
4. **Log everything** - Proper logging saved me hours of debugging.
5. **Cost vs. quality** - DeepSeek's output was more useful despite the small cost.

The fixed script has been running smoothly on my production servers for a month now. It's not enterprise-grade (you'd likely want to add proper rate limiting and TLS verification for that), but it does solve my immediate problem.
