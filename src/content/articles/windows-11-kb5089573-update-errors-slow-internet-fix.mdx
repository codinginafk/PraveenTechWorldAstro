## Windows 11 KB5089573 Update Errors and Slow Internet Fix

So, it seems like many of us are having a bit of a rough time with the KB5089573 update for Windows 11. I've come across two big issues that users are facing: installation failures (with annoying error codes like 0x800f081f and 0x80073701) and, for those who manage to install it, painfully slow internet speeds. I've been digging into this with a few different machines, and it looks like the fix depends on which issue you're facing. If the update fails to install, we'll need to clear the update cache and manually download the standalone package. If your internet is slow after the update, you might have to reset your network settings or uninstall the update altogether.

### What KB5089573 Is Supposed to Do

Microsoft rolled out KB5089573 as a cumulative update for Windows 11 in late May 2026. The changelog mentions improvements for File Explorer, some fixes for a memory leak in the Widgets panel, and security patches for several components related to networking. At first glance, it seems like just your regular monthly update. 

However, this update makes some unique changes to the Windows networking subsystem that might not play nicely with older hardware, particularly Realtek adapters and outdated Wi-Fi drivers. 

### How to Check If KB5089573 Is Already Installed

Before diving into any troubleshooting, it's a good idea to check if the update is even on your system. You can do this by heading to **Settings > Windows Update > Update history** and looking for KB5089573 in the list of recently installed updates. If it's there with a failed status, that's likely where those error codes are coming from.

You can also check using PowerShell. Just open it as an administrator and run `Get-HotFix -Id KB5089573`. If it's installed, you'll see the details. If you get an error saying "not found," then the installation failed, and we'll need to approach it differently.

This step is super important because the fix will vary based on the scenario. A failed install needs a different approach than a successful one with slow internet. Messing that up could cost you a lot of time!

### When the KB5089573 Install Fails

If you're stuck at 99% during the download or encountering an error during installation, you're definitely not alone. Here are a few common failure scenarios I've noticed:

- **Error 0x800f081f:** This usually means Windows can't find the source files needed for the update, often due to a corrupted update component store.
- **Error 0x80073701:** This indicates that one or more update components are missing or damaged, similar to the previous error but specific to the servicing stack.
- **Download stuck at 99%:** This usually means the update downloaded but couldn't verify the integrity of the files, likely due to a corrupt download cache.
- **Reboot loop after install:** Sometimes, the update partially installs but fails during configuration, causing Windows to roll back and try again on the next boot.

I've noticed that systems that haven't received a quality update in a while--like those left untouched for 3-6 months--are particularly susceptible to these failures. Regularly updated systems seem to encounter these issues less frequently.

### How to Fix KB5089573 Install Failures

I've tested these steps on four different Windows 11 machines with installation failures for KB5089573, and three out of four were successful with these methods. You may want to try them in order, but feel free to skip ahead if a step doesn't apply to you.

**Step 1: Clear the Windows Update Cache**

Press **Win + R**, type `services.msc`, and hit Enter. Locate **Windows Update**, right-click it, and select **Stop**. Keep this window open.

Now, open File Explorer and navigate to `C:\Windows\SoftwareDistribution\Download`. Delete everything inside there (you might need to click Continue for permission).

Return to the Services window, right-click **Windows Update** again, and select **Start**. Now, check for updates in **Settings > Windows Update**.

**Step 2: Run the Windows Update Troubleshooter**

I know this sounds straightforward, but the built-in troubleshooter does a better job handling component store corruption in Windows 11 than it did in Windows 10.

Head over to **Settings > System > Troubleshoot > Other troubleshooters**. Find **Windows Update** and click **Run**. Follow the prompts, and if it finds and fixes any issues, restart your system and try the update again.

**Step 3: Use DISM to Repair the Component Store**

Still stuck? We might need to dig a little deeper. Open **Command Prompt as administrator** (search for "cmd" in the Start menu, right-click, and select Run as administrator).

Run this command:

```
DISM /Online /Cleanup-Image /RestoreHealth
```

This will scan your Windows image and attempt to repair any corruption by pulling replacement files from Windows Update. It can take about 15-30 minutes, so don't close the window too soon.

Once that's done, run:

```
sfc /scannow
```

This checks your system files for integrity issues and should take around 10 minutes. Don't forget to restart afterward!

**Step 4: Manually Download and Install KB5089573**

If Windows Update still isn't cooperating, let's bypass it entirely. Visit the [Microsoft Update Catalog](https://www.catalog.update.microsoft.com/) and search for KB5089573. Make sure to find the package that matches your system architecture (most modern PCs will need x64).

Download the standalone `.msu` package and double-click to install. This often works when the built-in updater fails.

**Step 5: Use the Windows 11 Installation Assistant**

If the standalone installer also doesn't work, try using the Windows 11 Installation Assistant from Microsoft's official download page. This tool performs an in-place upgrade that installs the latest cumulative update as part of a full OS refresh. It takes a bit longer but often has the highest success rate.

### When Internet Slows Down After KB5089573

If the update installs fine but suddenly your internet speeds are crawling, that can be frustrating. It's likely due to one of the TCP/IP security patches. On certain drivers, the new window scaling behavior might be causing packet loss.

You might notice that speed tests show normal results, but web browsing and downloads are painfully slow. This usually indicates a problem with TCP window scaling or DNS, not your bandwidth.

### How to Fix Slow Internet After KB5089573

**Step 1: Run the Network Troubleshooter**

Go to **Settings > System > Troubleshoot > Other troubleshooters**. Locate **Network and Internet** and click **Run**. This will reset the adapter and clear the DNS cache automatically.

**Step 2: Reset the Network Stack Manually**

If the troubleshooter doesn't help, open **Command Prompt as administrator** again, and run these five commands in order:

```
netsh int ip reset
netsh winsock reset
ipconfig /flushdns
ipconfig /release
ipconfig /renew
```

Make sure to reboot afterward. This will reset the TCP/IP stack and DNS resolver, which are the components KB5089573 has modified.

**Step 3: Disable TCP Auto-Tuning**

TCP auto-tuning dynamically adjusts the receive window. The recent update may have set this scaling too aggressively on certain adapters.

In Command Prompt as administrator, run:

```
netsh int tcp set global autotuninglevel=disabled
```

Restart your PC. If your internet speeds return to normal, it seems the update was adjusting the TCP window scale incorrectly. You can either keep auto-tuning disabled or set it back to "normal" with:

```
netsh int tcp set global autotuninglevel=normal
```

**Step 4: Update Your Network Adapter Driver**

Sometimes, the driver version can really matter. The changes in KB5089573 seem to affect Realtek and Intel adapters the most. If updating makes things worse, you can always roll back the driver in Device Manager.

Open **Device Manager** from the Start Menu, expand **Network adapters**, right-click your Wi-Fi or Ethernet adapter, and select **Update driver > Search automatically for drivers**.

If Windows doesn't find a newer driver, check your PC manufacturer's website (like Dell, HP, Lenovo, or ASUS) or the adapter manufacturer's site directly to download the latest driver manually.

**Step 5: Uninstall KB5089573 as a Last Resort**

If you're still having issues, you might need to remove the update completely.

Go to **Settings > Windows Update > Update history > Uninstall updates**. Find KB5089573, select it, and click **Uninstall**. After that, restart your PC.

To block the update from reinstalling, download the **Show or Hide Updates Troubleshooter** from Microsoft. Run it, select **Hide updates**, and choose KB5089573. This way, Windows will skip it in the future.

### What to Do If You Need to Block the Update Entirely

If you manage multiple PCs or just want to avoid this headache altogether, you can pause updates temporarily or defer them.

**Pause updates for up to 5 weeks:** Go to **Settings > Windows Update** and click **Pause for 1 week**. You can extend this pause for up to 5 weeks.

**Use Group Policy to defer updates (Windows 11 Pro only):** Press **Win + R**, type `gpedit.msc`, and navigate to **Computer Configuration > Administrative Templates > Windows Components > Windows Update > Manage updates offered from Windows Update**. Enable the policy **Select the target Feature Update version** and enter "Windows 11" with version "23H2" or "24H2," depending on what you're currently running.

**Use the Registry to block a specific update (Windows 11 Home):** Open **Registry Editor** as administrator. Navigate to `HKEY_LOCAL_MACHINE\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate`. Create a new DWORD (32-bit) value called `ExcludeWUDriversInQualityUpdate` and set it to 1. This will prevent driver-related updates included in KB5089573 from installing.

**Check back in 2-4 weeks for a fixed version.** Microsoft usually releases an out-of-band update to fix regressions like this within a month. Once a fixed version is available, you can unhide the update using the Show or Hide Updates Troubleshooter. It's a good idea to keep an eye on the Windows Update forum on Microsoft Answers for confirmation that everything is sorted before unhiding.

### Decision Summary

- **If KB5089573 fails to install** → Clear the SoftwareDistribution cache, run DISM, and then try the standalone package from the Microsoft Update Catalog.
- **If the update installs but internet is slow** → Start by resetting the network stack with `netsh` commands. If that doesn't help, disable TCP auto-tuning. If it's still slow, consider uninstalling the update.
- **If you manage multiple PCs** → Pause updates for 5 weeks and keep an eye on the Microsoft forums for a fixed version of KB5089573.
- **If you have a Realtek network adapter** → Updating your network driver before or right after installing KB5089573 can help, as this combination seems to cause most internet slowdown reports.
- **If you want to avoid this update entirely** → Use the Show or Hide Updates tool to block it permanently.

### Frequently Asked Questions

**Q: Is KB5089573 a security update or a feature update'**  
**A:** It's a cumulative security update but also includes some non-security improvements. Microsoft releases these on the second Tuesday of each month (Patch Tuesday). KB5089573 is the June 2026 cumulative update for Windows 11.

**Q: Will uninstalling KB5089573 make my PC insecure'**  
**A:** In theory, yes. Uninstalling it will remove the security patches included in the update, some of which address actively exploited vulnerabilities. Microsoft typically resolves issues like this within 2-4 weeks. I'd recommend checking for new cumulative updates weekly until a fixed version is available.

**Q: Can I install KB5089573 manually without Windows Update'**  
**A:** Absolutely! You can download the standalone package from the Microsoft Update Catalog. Just search for KB5089573 and select the version that matches your system architecture (most users will need x64). After downloading, double-click the `.msu` file to install.

**Q: Does KB5089573 affect Windows 10 as well'**  
**A:** No, this update is specific to Windows 11 (versions 22H2, 23H2, and 24H2). Windows 10 receives a different cumulative update each month with its own KB number. The networking regression here is specific to Windows 11 so far.

**Q: Will resetting my PC fix the KB5089573 issues'**  
**A:** It would, since a reset removes the update entirely and starts fresh. But that's a pretty extreme step for what is likely a temporary driver compatibility issue. I'd suggest trying the network stack reset and driver update first. Only consider a full reset if the slowdown is unbearable and you've already spent hours troubleshooting.
