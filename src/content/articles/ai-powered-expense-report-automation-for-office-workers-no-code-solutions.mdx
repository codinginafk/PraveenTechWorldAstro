## How Can AI Automate My Expense Reports Without Writing Code?  

Have you ever felt bogged down by the tedious process of managing expense reports? I know I have! This is actually what led me to explore AI solutions. I turned to DeepSeek to create a Python script that extracts data from receipt images, categorizes expenses, and formats everything into a neat CSV report. The cool part? It all starts just by dragging files into a folder. DeepSeek took care of about 80% of the code, allowing me to focus on structuring the workflow and fixing a few library conflicts.  

### The Problem: Expense Reports Were Killing My Productivity  

Let’s be real—spending two hours every Friday on tasks that could be automated is frustrating! I was stuck doing things like:  
1. Cropping receipt images  
2. Typing amounts into Excel  
3. Assigning categories (like Travel, Meals, etc.)  
4. Calculating totals  

To make matters worse, our finance team needed reports in a specific CSV format, including columns for Date, Vendor, Amount, and Category. I can't tell you how many times I transposed numbers (like typing $56.20 as $65.20), which meant I had to redo my work.  

### The AI Attempt: A One-Shot Prompt to DeepSeek  

So, I decided to give DeepSeek a shot with the following prompt:  

```  
Prompt:  
"Build a Python script that:  
1. Watches a 'receipts' folder for new image files (JPG/PNG)  
2. Uses OCR to extract text from each image  
3. Identifies the vendor name, date, and total amount  
4. Categorizes expenses as 'Travel', 'Meals', or 'Office Supplies' based on keywords  
5. Appends the data to a CSV with headers: Date,Vendor,Amount,Category  
6. Runs automatically when new files appear  

Use pytesseract for OCR and watchdog for file monitoring.  
Include error handling for bad OCR reads."  
```  

**First Output:**  
DeepSeek generated a script with 47 lines, utilizing:  
- `pytesseract` for OCR  
- `watchdog` for folder monitoring  
- Regex patterns to identify amounts and dates  

**Where It Worked:**  
- The folder watcher logic was on point  
- The CSV generation handled proper escaping just fine  

### Where It Broke: 3 Critical Failures  

Unfortunately, things weren’t all smooth sailing. Here are a few hiccups I ran into:  

1. **Library Hell**  
   The script needed `pytesseract`, but it failed because Tesseract OCR wasn’t installed system-wide. I got an error that said:  
   ```  
   TesseractNotFoundError: tesseract is not installed or it's not in your PATH  
   ```  

2. **Vendor Detection Was Garbage**  
   When I tested it with a Starbucks receipt, it spat out:  
   ```  
   Vendor: STARBUCKS INC 12345  
   Date: 2024-02-30 (invalid date)  
   Amount: $12.SO (it mistook '5' for 'S')  
   ```  

3. **False Positives**  
   A screenshot of a Slack conversation triggered the script because it contained a dollar sign ($). Yikes!  

### What I Had to Fix  

**1. The OCR Pipeline**  
I had to switch to EasyOCR since pytesseract struggled with thermal receipts. Here’s how I adjusted the code:  
```python  
# Before (pytesseract)  
text = pytesseract.image_to_string(image)  

# After (EasyOCR)  
reader = easyocr.Reader(['en'])  
results = reader.readtext(image, detail=0)  
```  

**2. Date Validation**  
I added a function to reject impossible dates (like February 30th):  
```python  
from datetime import datetime  
def validate_date(date_str):  
    try:  
        datetime.strptime(date_str, '%Y-%m-%d')  
        return True  
    except ValueError:  
        return False  
```  

**3. Vendor Cleanup**  
To improve vendor detection, I used a predefined vendor list with fuzzy matching:  
```python  
KNOWN_VENDORS = ['Starbucks', 'Uber', 'Amazon Web Services']  

def match_vendor(raw_text):  
    for vendor in KNOWN_VENDORS:  
        if vendor.lower() in raw_text.lower():  
            return vendor  
    return "Unknown"  
```  

### The Working Result  

In the end, my script turned out to be pretty handy and included these features:  
- **Drag-and-drop processing**: Just drop receipts into `~/expense_reports/receipts`  
- **Human review**: It flags uncertain entries in yellow for manual confirmation  
- **Audit trail**: It logs all actions in `processing.log`  

**Sample Output CSV:**  
```  
Date,Vendor,Amount,Category  
2024-03-15,Starbucks,12.50,Meals  
2024-03-16,Uber,24.30,Travel  
```  

**Execution Time:**  
- It takes about 0.8 seconds per image (I tested it on 20 receipts)  
- I noticed a 95% accuracy on typed receipts and about 70% on handwritten ones  

### Key Takeaways  

1. **AI Struggles with Edge Cases Without Human Intervention**  
   I found that DeepSeek didn’t quite handle thermal receipt fade or handwritten notes. I had to integrate manual review steps to make it effective.  

2. **Effective Prompt Engineering Requires Detailed Specifications**  
   The more specific my prompt was (like saying "include a 10% confidence threshold for OCR results"), the better the output I received.  

3. **The Importance of Choosing the Right Libraries**  
   In my case, EasyOCR performed better than pytesseract, but it did add about 300MB to the deployment size.  

### The Exact Prompt  

Here’s the refined prompt that led to the final working version:  

```  
"Build a Python 3 script that:  
1. Uses EasyOCR instead of pytesseract for better receipt readability  
2. Watches a folder for new JPG/PNG files with watchdog  
3. Extracts:  
   - Date (must be valid, skip if invalid)  
   - Amount (must match regex ^\$?\d+\.\d{2}$)  
   - Vendor (fuzzy match against a hardcoded list)  
4. Categorizes expenses using these rules:  
   - 'Travel' if vendor is Uber/Lyft or contains 'taxi'  
   - 'Meals' if vendor is a restaurant  
   - 'Office' if vendor is Amazon/Staples  
5. Outputs to CSV with error logging  
6. Skips images where confidence is below 70%  

Include a function to manually review uncertain entries before CSV export."  
```

## FAQ  

**Q: Can this handle multi-currency receipts?**  
A: Not just yet. The current regex only catches USD ($). You’d need to tweak the amount detection pattern for that.  

**Q: What about receipts in other languages?**  
A: EasyOCR supports over 80 languages! You just need to change the Reader initialization to something like `['en','es','fr']` as necessary.  

**Q: Is this GDPR compliant?**  
A: Unfortunately, no—the script stores raw receipt images alongside processed data. You would need to implement an auto-deletion feature for images after processing.  

**Q: Can I run this on my phone?**  
A: You can only run it via Termux (which is a Linux environment for Android). For iOS, you’d likely need a cloud trigger like AWS Lambda.  

**Q: How much did this cost to build?**  
A: It was free in terms of tools (all open-source), but I spent about 3 hours on prompt tuning and debugging.
