Have you ever opened Google Search Console only to see a frustrating red error next to your sitemap? Maybe it says 'Couldn’t fetch,' 'Has errors,' or 'URLs not indexed.' It’s a bit disheartening, especially when your website is up and running perfectly fine on your end. 

I’ve been there too, having submitted sitemaps for various sites and encountering all sorts of errors. Let me break down each error type you might come across in Google Search Console, explain what they mean, and share some straightforward solutions to get things back on track.

## What Do These Errors Mean?

Essentially, sitemap errors in Google Search Console indicate that Google tried to read your sitemap file but ran into some issues. Each error type has its own cause and solution. For instance, 'Couldn’t fetch' means Google can’t access the file, 'Has errors' indicates there’s something wrong with the file format, and 'URLs not indexed' means Google found your sitemap but opted not to include certain pages. Don't worry—I’ll guide you through fixing each one!

## Error 1: Couldn't Fetch

This is probably the most common error you'll see. It happens when Googlebot tries to download your sitemap file but can’t get a response. 

Here’s what to check:

1. **Sitemap URL**: Type your sitemap URL directly into your browser. For example, go to `https://www.yoursite.com/sitemap.xml`. If it shows a 404 page, your sitemap file isn't where it should be. You might need to move it or update the path in GSC.

2. **Server Access**: If your sitemap loads in your browser but GSC still shows 'Couldn't fetch,' there might be a server block. Googlebot could be blocked by your firewall, CDN, or server configurations. Check your server logs for requests from Googlebot’s IP ranges. If you use a service like Cloudflare, make sure it isn’t blocking Googlebot.

3. **Password Protection**: A common issue is if your sitemap is behind a password-protected site. If your site requires a login, Googlebot can’t access it. Make sure your sitemap is publicly accessible.

## Error 2: Has Errors

When GSC says 'Has errors' but isn’t specific, it usually means there’s a formatting issue with your sitemap file. Open the sitemap URL in your browser to view the raw XML.

Here are a few common formatting errors to look out for:

- **Missing Closing Tags**: Every `<url>` tag needs a matching `</url>` closing tag.
- **Invalid XML Characters**: Characters like & < > " and ' must be properly escaped (e.g., `&amp;`, `&lt;`, etc.).
- **Wrong Namespace**: Your sitemap XML should start with: `xmlns="http://www.sitemaps.org/schemas/sitemap/0.9"`.
- **Encoding Issues**: Ensure your sitemap is saved as UTF-8 without BOM.

You can use an XML validator like validator.w3.org to check your sitemap file. Just paste your sitemap URL in there, and it’ll help you pinpoint the error.

## Error 3: URLs Not Indexed

This one can be a bit tricky! If your sitemap submitted without any issues, but the Index Coverage report shows that some URLs aren’t indexed, Google is evaluating those URLs for indexing.

Common reasons for this include low-quality pages, duplicates, or pages blocked by noindex tags. 

Use the URL Inspection tool in GSC for each non-indexed URL to find out why it’s not indexed. Some possible causes are:

- **Noindex Tag**: If a page has a noindex tag, remove it and request indexing.
- **Duplicate Pages**: If it’s a duplicate, add a canonical URL pointing to the original.
- **Blocked by robots.txt**: Check your robots.txt file and ensure the path is allowed. You can read more about this in our [guide to sitemap URLs blocked by robots.txt](/blog/sitemap-urls-blocked-by-robotstxt-clean-and-resubmit).
- **New Pages**: If a page is too new, Google may not have crawled it yet. You can request indexing manually.

## Error 4: Sitemap is Too Large

Google has some specific limits on sitemap files: each sitemap can contain a maximum of 50,000 URLs and be no larger than 50MB uncompressed (or 10MB when gzipped). If your sitemap exceeds this, Google won’t process it.

To fix this, you might need to split your sitemap into smaller files and manage them with a sitemap index file. For example, create separate sitemaps for blog posts, product pages, and category pages. Then, create a sitemap index file that references all of them like this:

```xml
<sitemapindex xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
  <sitemap>
    <loc>https://www.yoursite.com/sitemap-posts.xml</loc>
  </sitemap>
  <sitemap>
    <loc>https://www.yoursite.com/sitemap-pages.xml</loc>
  </sitemap>
</sitemapindex>
```

Just submit the sitemap index file to GSC instead of each individual file.

## Error 5: Sitemap Contains Noindex URLs

This error is a bit of a logic issue. If your sitemap includes URLs that have a noindex robots meta tag, it sends mixed signals to Google. You're telling Google to include and exclude the page at the same time.

Go through your sitemap and remove any URLs that have a noindex directive. After that, wait for Google to recrawl the sitemap, and the warnings should clear up.

## Error 6: Redirect in Sitemap

Every URL in your sitemap should return a 200 status code. If any URL redirects (like 301 or 302) to somewhere else, GSC will flag it. 

You can either update your sitemap with the final destination URL or fix the redirect so that the original URL returns a 200 status. The sitemap should always reflect the canonical version of each URL.

## Error 7: Sitemap URL is Blocked by robots.txt

Before Googlebot attempts to crawl your sitemap, it checks your robots.txt file. If it’s blocking the sitemap path, Googlebot can’t access it.

Check your robots.txt file at `https://www.yoursite.com/robots.txt` to ensure it’s not disallowing any paths related to your sitemap. You can also explicitly declare your sitemap by adding a line like this:

```plaintext
Sitemap: https://www.yoursite.com/sitemap.xml
```

This gives Google an extra way to find your sitemap.

## Error 8: Sitemap Dates in the Future

Each `<lastmod>` date in your sitemap should be in the past. If your sitemap generator accidentally sets future dates, it might confuse Google about when the page was last updated.

Make sure your sitemap generator is set up to use actual last-modified dates. If you’re unsure, it’s okay to omit the `<lastmod>` tag altogether, as GSC doesn’t require it.

## Quick Recap

- For 'Couldn't fetch': Check the file path, server access, and firewall settings.
- For 'Has errors': Validate the XML structure with an online validator.
- For 'URLs not indexed': Use the URL Inspection tool for detailed insights.
- If your sitemap was never submitted: Go ahead and submit it now from the Sitemaps section in GSC.

## When Do These Fixes Work?

These solutions typically work when the issue is with the sitemap file itself, server configuration, or the GSC property setup. If you follow these steps carefully for the specific error, your sitemap should show a Success status in just a few days.

## When Might It Not Work?

If you fix the error and resubmit but GSC still shows the same issue after about a week, there might be a bug on Google’s end. It’s rare but does happen occasionally. In that case, try removing the sitemap entirely, wait 24 hours, and then submit it as a new sitemap.
