## Why This Matters

Let's be real. When you're applying for a job or a freelance gig, your resume is just a PDF. A portfolio website is where you bring that resume to life. It’s your digital handshake.

Hiring a web developer costs a fortune, though. Using platforms like Wix or Squarespace means monthly fees. For many of us, that’s just not in the budget—especially when we’re starting out. That’s the problem. You need a way to showcase your work without spending a dime or learning complex code from scratch.

## What You Will Learn

In this guide, you’re going to walk away with a brand-new, live website. Specifically, you’ll learn how to:

*   Set up a GitHub repository to host your site for free.
*   Create a simple, clean website using HTML and basic CSS.
*   Push your local files to GitHub so they appear online.
*   Customize your site’s look with a pre-made template.
*   Connect a custom domain name if you choose to later.
*   Update your portfolio anytime with just a few clicks.

## Step-by-Step Guide

### Step 1: Create Your GitHub Home

First, sign up for a free account at GitHub.com if you don't have one. It's like making a social media profile, but for your code and files.

Once you're in, click the "+" icon in the top right and select "New repository." Name it exactly like this: `yourusername.github.io`. This special name tells GitHub to make this repository into a website. Check the box that says "Add a README file" and hit create repository. Your website now has an address: `yourusername.github.io`.

### Step 2: Build Your First Webpage

You don't need a fancy text editor. You can use the one built right into GitHub. In your new repository, click the "Add file" button and choose "Create new file."

Name the file `index.html`. This is your homepage. GitHub gives you a simple editor. Copy and paste this code into it:

```html
<!DOCTYPE html>
<html>
<head>
    <title>My Portfolio</title>
</head>
<body>
    <h1>Hi, I'm [Your Name]</h1>
    <p>Welcome to my corner of the internet.</p>
</body>
</html>
```

Change `[Your Name]` to your actual name. Scroll down, write a commit message like "First homepage," and click "Commit changes." Wait two minutes. Visit `yourusername.github.io` in your browser. You did it. Your website is live.

### Step 3: Make It Look Professional (The Easy Way)

The plain white page works, but it's not very impressive. Let's fix that in 60 seconds. We'll use a free template.

Find a template you like from sites like HTML5 UP or Start Bootstrap. Download the ZIP file. Extract it on your computer.

Go back to your GitHub repository. Click "Add file" then "Upload files." Drag the entire folder of files from the template you downloaded into the upload box. Commit the changes. Give GitHub a minute to update. Now refresh your `yourusername.github.io` page. Boom. You've got a sleek, modern portfolio site.

### Step 4: Customize Your Content

This is the most important part. Open the `index.html` file from the uploaded template by clicking on it in your repository, then click the pencil icon to edit.

Don't be scared by the code. You only need to look for the text between the `< >` tags. Change the headings, paragraphs, and links to match your own work. If you have project images, you'll need to replace the image file names in the code with your own. Just keep your file names simple, like `project1.jpg`. You can upload your images the same way you uploaded the template files.

## Tips for Best Results

1.  **Start simple.** Don't try to build a multi-page, interactive masterpiece on day one. A clean, one-page site that loads fast is better than a slow, broken complex one.
2.  **Keep your file names clean.** Use `about.html`, not `About_Me_Page (2).html`. It makes your links work properly.
3.  **Mobile is not optional.** Over 60% of web traffic is on phones. Before you show anyone your site, check it on your own phone. If text is tiny or buttons are impossible to tap, fix it.
4.  **Update it regularly.** A portfolio with a "latest project" from 2021 looks abandoned. Add a new project or skill every month. It shows you're active.
5.  **Add a simple contact method.** Put a working email link or a link to your LinkedIn in the header or footer. Don't make people hunt for it.

## Frequently Asked Questions

**1. Is this truly free?**
Yes, 100% free. GitHub Pages offers free static site hosting. As long as your site is public, you won't pay anything. You only pay for a custom domain name if you want one, which is about $10-15 per year.

**2. Do I need to know how to code?**
No, not really. You can get by with copying, pasting, and changing text. For a basic portfolio, that’s enough. As you get more comfortable, you can learn a little HTML and CSS to make more changes.

**3. Can I use my own .com name instead of .github.io?**
Absolutely. You can buy a domain from a site like Namecheap or Google Domains. Then, you follow GitHub’s instructions to connect it to your site. The whole process takes about 15 minutes.
